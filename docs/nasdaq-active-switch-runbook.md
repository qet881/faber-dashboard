# 나스닥100 슬롯 패시브 → 액티브 전환 런북

> 이 문서는 faber-dashboard(`app.py`)를 다루는 **누구든**(다른 Claude Code 포함) 따라할 수 있도록
> 자립형으로 작성됨. 사용자가 "나스닥 액티브로 갈아탔다"고 하면 이 문서대로 처리한다.

## 1. 배경

- 전략: 해남 A. 나스닥100 신호가 ON이면 나스닥100 슬롯에 비중이 배분된다.
- **기본 집행 상품은 액티브 ETF 2개**: `TIME 나스닥100액티브`(티커 426030) 50% + `KoAct 나스닥100액티브`(티커 0015B0) 50%.
  - 이 분배는 `_nasdaq_active_execution_targets()` / `expand_haenam_execution_weights()`가 담당.
- **그러나 2026-05 리밸런싱부터는 액티브 ETF의 괴리율(프리미엄)이 높아, 사용자가 실제로는
  패시브 `미국나스닥100`(티커 133690)을 매수**했다.
- 계획: **액티브 괴리율이 0.3% 미만으로 떨어지면 액티브로 갈아탄다.**

## 2. 무엇이 영향을 받고, 무엇이 안 받나 (중요)

대시보드 "📅 이번 달 성과" 섹션에는 두 블록이 있다:

1. **공식 성과: 실제 계좌 기준** — `공식 손익 = 현재 운용자산 − 전월말 기준 총자산 − 순외부현금흐름`.
   - 계좌 **총액**만으로 계산. 나스닥을 패시브로 사든 액티브로 사든 **이 숫자는 1원도 안 바뀐다.**
   - 월말 정산값(원장)을 그대로 쓴다. → 전환 작업과 **무관**.
2. **참고: 자산별 가격변동 추정** — 전략 비중 × 각 자산의 (현재가/리밸일가 − 1).
   - 여기 나스닥 칸이 패시브로 보이느냐 액티브로 보이느냐가 이 전환 작업의 **유일한 대상**.
   - 가격은 데이터피드의 **ETF 시장 종가**를 쓴다(리밸일 시점 종가 → 현재 시점 종가).
     **사용자의 실제 체결가가 아니다.** 그래서 전환에 필요한 정보는 '날짜'뿐이다.

## 3. 동작 원리

- 상수: `HAENAM_NASDAQ_ACTIVE_SWITCH_DATE` (`app.py`, `HAENAM_KOACT_NAME` 정의 바로 아래).
- "이번 달 성과" 섹션(`expand_haenam_execution_weights(...)` 호출 직후)에서 다음을 한다:
  - `HAENAM_NASDAQ_ACTIVE_SWITCH_DATE`가 `None`이거나, `rebal_date < SWITCH_DATE`이면
    → 해당 월 나스닥 비중을 `TIME`+`KoAct`에서 **패시브 `미국나스닥100`으로 합쳐서** 표시.
  - 그렇지 않으면(= `rebal_date >= SWITCH_DATE`) → 액티브(TIME/KoAct) 그대로 표시.
- `rebal_date` = **그 달 성과의 기준일 = 전월 마지막 거래일**.

## 4. 전환 시 처리 절차 (사용자가 액티브로 갈아탔다고 할 때)

1. 사용자에게 **전환(매수)한 날짜**를 받는다. (체결가/수량도 받으면 좋지만 계산엔 안 쓰임 — 저널 기록용.)
2. `app.py`의 `HAENAM_NASDAQ_ACTIVE_SWITCH_DATE = None` 을
   `HAENAM_NASDAQ_ACTIVE_SWITCH_DATE = pd.Timestamp('YYYY-MM-DD')` 로 바꾼다.
   - 값 = 사용자가 액티브를 매수한 날(리밸런싱일).
   - **예시**: 2026-06-30 종가에 액티브로 갈아탔다 → `pd.Timestamp('2026-06-30')`.
     - 6월 성과(rebal_date=2026-05-29 < 2026-06-30) → **패시브**로 표시 (6월 내내 패시브 보유했으므로 맞음).
     - 7월 성과(rebal_date=2026-06-30 >= 2026-06-30) → **액티브**로 표시. ✅
3. `python -m py_compile app.py` 로 문법 확인.
4. 커밋/푸시 (사용자가 원할 때).
5. **저널 기록**: `.codex-private/journal.md` 또는 `monthly_ledger.md`에 전환일·체결가·사유(괴리율 0.3% 미만 도달)를 남긴다.
   - `.codex-private`는 `.gitignore`된 **별도 git repo**(remote: faber-investment-memory)다.
     `git -C .codex-private add/commit/push` 로 따로 관리한다.

### 월 중간에 갈아탄 경우
참고 섹션은 한 달을 한 상품으로 가정하는 근사치다(캡션에도 "실제와 차이 있을 수 있음" 명시).
월 중간 전환이면 **전환한 달은 패시브로, 다음 달부터 액티브로** 표시된다(위 규칙 그대로). 근사치라 이대로 둔다.

- 2026-05 정산: 월초 283,565,328 → 월말 312,411,996, 공식 손익 +28,846,668 (+10.17%), 5월 현금흐름 0.
- 코스피200 슬롯은 이미 삼성전자(005930)/SK하이닉스(000660) 개별주로 집행 중.
- 월말 정산 기준: KRX 정규장 종가(15:30 확정). 개별주 NXT 시간외(15:30~20:00)는 제외.
